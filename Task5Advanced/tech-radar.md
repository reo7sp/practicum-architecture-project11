# Tech Radar

## Методы и паттерны

| Технология / паттерн | Статус | Зачем |
|---|---|---|
| Data Mesh | Adopt | Базовый подход для разделения данных по доменам и развития data products |
| Event-Driven Architecture | Adopt | Основа для слабосвязанных интеграций и near-real-time обработки |
| Self-Service BI | Adopt | Целевая модель доступа бизнеса к витринам и метрикам |
| Domain Data Products | Adopt | Стандарт публикации данных доменами |
| Lakehouse подход | Trial | Подходит для поэтапного перехода к Iceberg-based аналитическому слою |
| Data Contracts | Assess | Формализация обязательств между доменами |

## Платформы

| Технология / паттерн | Статус | Зачем |
|---|---|---|
| RBAC / ABAC | Adopt | Контроль доступа к медицинским и финансовым данным |
| Kubernetes | Trial | Унификация запуска новых сервисов и data workloads |
| Object Storage Data Lake | Trial | Дешёвое масштабируемое хранение сырых и подготовленных данных |
| Stream Processing | Trial | Переход от batch к near-real-time витринам |
| Power BI как основной BI-слой | Hold | Можно сохранять временно, но не как единственную стратегическую платформу |
| SQL Server 2008 DWH как центр логики | Hold | Legacy, подлежит выводу из критического контура |
| Camel ESB как основной интеграционный слой | Hold | Допустим только как временный bridge |
| PowerBuilder | Hold | Не развивать, постепенно выводить |

## Инструменты

| Технология / паттерн | Статус | Зачем |
|---|---|---|
| Schema Registry | Adopt | Контроль контрактов событий |
| DLQ | Adopt | Надёжная обработка ошибок в событиях |
| Data Catalog | Adopt | Поиск, описание и lineage data products |
| CDC | Trial | Поэтапная миграция с legacy и публикация изменений из старых систем |
| Feature Store / MLOps platform | Assess | Может понадобиться при масштабировании AI-направления |
| GraphQL для self-service слоя | Assess | Может упростить доступ к нескольким data products |
